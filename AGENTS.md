# 项目执行规则（C++版本）

## 核心要求
**分阶段执行，每个阶段完成后主动停止运行，必须运行测试并展示结果给用户确认，才能进入下一阶段。禁止在一个对话中执行多个阶段任务，避免上下文过大导致的问题。**

## 阶段划分（迭代式开发）
本项目采用小步迭代开发，共分为9个迭代阶段，按顺序执行：
1. 迭代0：项目基础搭建（最小可运行demo）
2. 迭代1：电池信息采集显示
3. 迭代2：硬件信息页面
4. 迭代3：页面框架完善
5. 迭代4：功耗监控页面（root权限功能）
6. 迭代5：温度监控页面
7. 迭代6：进程能耗页面
8. 迭代7：系统状态页面
9. 迭代8：概览页面
10. 迭代9：优化和打包发布

## 执行规则
### 0. 开发准则（必须严格遵守）
所有开发工作必须遵循以下原则：
1. **思考先行**：编码前深入思考，明确陈述假设，权衡利弊。遇到歧义时立即询问澄清，绝不静默猜测。
2. **最小代码**：仅编写解决当前问题所需的最少量代码，严格避免预测性功能、未请求的抽象或提前配置。
3. **精准修改**：编辑现有代码时进行高度精准的手术式修改，仅更新满足需求所需的确切行，完美保持现有风格。除非变更直接导致导入或变量失效，否则完全不改动相邻的正常代码。
4. **目标驱动**：每项任务都通过目标导向的方式执行，分解为清晰的分步计划并制定明确的成功标准，例如先编写可复现的失败测试，独立循环验证直至严格满足特定目标。
5. **文档同步**：阶段或功能开发完成后，必须同步更新PLAN.md（进度标记和完成情况）和ARCHITECTURE.md（架构和结构变更），确保文档与代码始终保持一致。
6. **强制测试**：每个迭代/阶段完成后，必须立即运行程序测试所有功能，截图/录屏展示运行效果，确认100%没问题再提交。

### 1. 新对话启动
- 每次开启新对话时，首先读取以下核心文档获取上下文：
  - `PLAN.md`：项目计划和阶段划分
  - `AGENTS.md`：执行规则和要求（本文件）
  - `ARCHITECTURE.md`：架构设计和目录结构说明
- 确认当前所处的阶段和需要执行的任务

### 1. 阶段启动
- 每个阶段开始前，用户明确指定要执行的阶段
- 严格按照PLAN.md中定义的阶段目标和任务范围执行，不超出当前阶段范围
- 执行过程中保持上下文聚焦，仅处理当前阶段相关的问题

### 2. 阶段执行
- 仅执行当前阶段定义的任务，不提前进行下一阶段的工作
- 遇到问题优先在当前阶段范围内解决，如涉及跨阶段问题，停止并告知用户
- 定期同步执行进度，关键节点及时反馈
- **代码质量要求**：
  - 每次写完代码后，必须使用代码格式化工具（如black、ruff等）进行格式化，避免手动格式化失误
  - 优先使用工具完成重复性工作（格式化、语法检查、测试执行等），不依赖手动处理
- **测试要求**：
  - 每个功能模块开发完成后必须编写对应的测试用例
  - 所有代码在提交前必须通过测试验证
  - 测试过程优先使用自动化测试工具执行
- **文件管理要求**：
  - 所有代码和文档按照ARCHITECTURE.md中定义的目录结构存放
  - 临时文件统一存放在临时目录，不与正式代码混放

### 3. 阶段完成
- **清理要求**：阶段完成后必须清理所有无用和临时文件
  - 清理范围：测试临时数据、调试日志、临时备份文件、编译中间文件等
  - 保留范围：源代码、文档、测试用例、样例数据、必要的配置文件
  - 清理完成后保持目录结构完整，符合ARCHITECTURE.md定义
- 完成当前阶段所有定义的任务后，立即停止执行
- 主动向用户汇报阶段完成情况，包括：
  - 完成的任务清单
  - 交付的产物列表
  - **测试运行情况和结果展示**（包括成功用例、失败用例、覆盖率等）
  - **功能演示截图或录屏**（可交互功能需要展示实际运行效果）
  - 临时文件清理情况说明
  - 遇到的问题和解决方案
- 等待用户确认和下一阶段指令，不自动开始后续工作

### 4. 阶段切换
- 用户确认当前阶段完成后，再开始下一阶段的工作
- 新阶段开始前，重新梳理上下文，确保聚焦新阶段的目标

## 上下文管理规则
- 每个阶段的对话保持独立，避免携带过多历史上下文
- 需要历史信息时，主动读取相关文件（PLAN.md、ARCHITECTURE.md、代码文件、文档等），不依赖对话历史
- 每个阶段的产出物都写入文件永久存储，不依赖对话上下文保存重要信息
- 架构变更需要及时更新ARCHITECTURE.md，保持文档与实际结构一致

## 阶段验收标准
每个阶段完成后需要满足以下要求，经用户确认验收通过后，方可进入下一阶段：
1. 完成PLAN.md中定义的该阶段所有任务
2. 交付物齐全，符合要求
3. 所有测试用例通过
4. 代码已使用工具格式化，符合代码规范
5. 无用和临时文件已清理完毕
6. 相关文档已同步更新
